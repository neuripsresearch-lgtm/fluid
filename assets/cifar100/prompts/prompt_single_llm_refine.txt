You are an expert taxonomist and a hierarchical data architect.

Your goal is to improve the classification performance of a model by refining its underlying class hierarchy (a JSON tree). You will be given the current tree, performance metrics, and a list of specific mistakes the model is making.

**You must analyze these mistakes and DIRECTLY output a corrected version of the JSON tree.**

---
### Input Data

1.  **Current Hierarchy**: The current JSON structure.
2.  **Performance Metrics**: Data on how well the model is performing.
3.  **Misclassifications**: A list of specific errors, e.g., `(True: 'seal', Predicted: 'otter')`.

### Your Task

1.  **Analyze the Errors**: Look for semantic patterns in the misclassifications.
    * If 'seal' is confused with 'otter', are they close enough in the tree? If they are in different subtrees (e.g., one in 'fish' and one in 'mammal'), you must move them closer (e.g., under 'aquatic_mammals').
    * If two distinct classes are merged too early, separate them.
    * **Ignore visual confusion**: If 'leopard' is confused with 'jaguar', this is visual similarity. Do not break the hierarchy for this. Only fix logical/semantic structural issues.

2.  **Refine the Tree**: Based on your analysis, modify the JSON structure to fix these issues. You can:
    * Move nodes to better parents.
    * Create new intermediate parent nodes (e.g., create 'feline' to group cats).
    * Rename nodes for clarity.

### Constraints (CRITICAL)

* **Output Format**: You must output **ONLY the final, valid JSON object**. Do not output instructions.
* **Leaf Integrity**: The tree MUST contain exactly the same 100 leaf nodes (classes) as the input. **Do not add, remove, or rename leaf nodes.**
* **Structure**: Every node must have a `name`, `children` (list), and a `justification` string explaining the grouping.
* **Justification**: You MUST rewrite the `justification` field for any node you modify to accurately reflect its new children.
