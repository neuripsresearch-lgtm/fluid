### Execution Protocol

You must adhere to the following strict protocol:

1.  **Sequential Execution**: Instructions in the `edit_instructions` list are **ordered and stateful**. You must execute them in the sequence provided. The state of the tree after instruction `N` is the input for instruction `N+1`.

2.  **Node Path Syntax**: Node paths are strings with `.` as a delimiter, navigating from the root. For example, `entity.living_organism.animal` refers to the "animal" node. You must parse these paths to locate the correct nodes.

3.  **Invalid Instruction Handling**: If an instruction is impossible to execute (e.g., a `source_parent_path` does not exist in the tree's current state, or a `node_to_move` is not found), you must **ignore that single instruction** and proceed to the next one. Do not halt execution.


**Note that if the tree provided to you does not have names to the internal nodes then make sure to give them new appropriate and justified names by yourself accordingly while giving the new tree. ### Critical Mandates

These rules are non-negotiable and essential for a correct transformation.

1.  **Justification Coherency (Mandatory)**: This is your most critical task beyond structural changes.

      * After any operation that changes a parent node's `children` array (e.g., `MOVE_NODES`, `CREATE_AND_MOVE`), you **MUST** rewrite the `justification` string for **ALL affected parent nodes**.
      * For a `MOVE` operation, this includes both the **source parent** (which lost children) and the **destination parent** (which gained children).
      * The new justification must accurately and concisely describe the new set of children.

2.  **Structural and Content Integrity**:

      * The final output must be a single, valid JSON object that conforms to the original schema (`name`, `justification`, `children`).

3.  **ALL OF THE LEAF NODES SHOULD MATCH THE INITIAL LEAF NODES EXACTLY. NO EXTRA LEAF NODES SHOULD BE ADDED, NO EXISTING LEAF NODES SHOULD BE DELETED.**
      
      * There might be a case where all of the children of an internal node are moved somewhere. Make sure that that node is removed from the tree. The only leaf nodes should be the ones that are already leaf nodes, and they must not be removed.
